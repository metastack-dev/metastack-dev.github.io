# Приватный лендинг с деплоем на metastack-dev.github.io

**Дата:** 2026-07-03
**Статус:** утверждён

## Цель

Получить сайт на домене `https://metastack-dev.github.io`, не раскрывая исходный код лендинга. GitHub Pages не умеет проксировать (только статика), поэтому вместо «прокси» используется схема: исходники в приватном репо, в публичный репо CI пушит только собранные артефакты.

В рамках этой работы создаётся **только основа** (скаффолдинг + пайплайн + плейсхолдер-страница), не сам лендинг.

## Архитектура

Два репозитория:

- **`metastack-dev/landing` (приватный)** — исходники. Next.js (App Router, TypeScript) со статическим экспортом (`output: 'export'`), билд в `out/`. Плейсхолдер `app/page.tsx`.
- **`metastack-dev/metastack-dev.github.io` (публичный, этот)** — принимает только содержимое `out/`. Публично видны лишь минифицированный билд и коммиты от бота.

## Пайплайн деплоя

GitHub Actions workflow в приватном репо:

- **Триггер:** push в `main`.
- **Шаги:** checkout → setup-node → `npm ci` → `npm run build` → деплой `out/` в ветку `main` публичного репо через `peaceiris/actions-gh-pages` с `external_repository`.
- **`.nojekyll`** обязателен в корне деплоя: Next.js кладёт ассеты в `_next/`, а Jekyll игнорирует папки с подчёркиванием.

## Аутентификация

**Deploy key** (SSH-пара):
- приватная часть — секрет `ACTIONS_DEPLOY_KEY` в репо `landing`;
- публичная часть — deploy key с правом записи в `metastack-dev.github.io`.

Узко скоуплен на один репо — безопаснее PAT.

## Конфигурация Next.js

- `output: 'export'` в `next.config.ts`;
- `images: { unoptimized: true }` (оптимизация изображений требует сервер);
- никаких серверных фич (API routes, SSR) — только статика.

## Проверка

1. Push плейсхолдера в `main` приватного репо.
2. Workflow зелёный.
3. `https://metastack-dev.github.io` открывается и показывает плейсхолдер.
4. В публичном репо — только билд-артефакты, без исходников.

## Вне скоупа

- Дизайн и контент лендинга.
- Кастомный домен.
- Аналитика, формы, любой бэкенд.
